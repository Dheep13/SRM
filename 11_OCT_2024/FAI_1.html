1. State Space Search

Question 1.1 (15 Marks)

Given three jugs with capacities of 12 liters, 8 liters, and 3 liters respectively, with the 12-liter jug full and others empty:

a) Formulate this as a state space search problem to get exactly 6 liters in the 12-liter jug. (5 marks)

b) Show the first three valid moves and resulting states. (5 marks)

c) What search strategy would be most efficient for this problem and why? (5 marks)

Solution 1.1:

a) Problem Formulation (5 marks)

  • State Space: (x,y,z) where:
    • x = water in 12L jug
    • y = water in 8L jug
    • z = water in 3L jug
  • Initial State: (12,0,0)
  • Goal State: (6,*,*) where * can be any valid value
  • Operators: Pour from any jug to another

b) First Three Valid Moves (5 marks)

Initial: (12,0,0)
Move 1: Pour 12L→8L = (4,8,0)
Move 2: Pour 8L→3L = (4,5,3)
Move 3: Pour 3L→12L = (7,5,0)

c) Search Strategy (5 marks)

  • BFS (Breadth-First Search) would be most efficient because:
    • State space is finite and not very deep
    • Need to find shortest solution path
    • All moves have equal cost
    • Guarantees optimal solution

Question 1.2 (15 Marks)

Consider the 8-puzzle problem with the following configuration:

Initial State:     Goal State:
2 8 3             1 2 3
1 6 4             8   4
7   5             7 6 5

a) Define the heuristic function for this problem. (5 marks)

b) Calculate the Manhattan distance for the initial state. (5 marks)

c) Show the first two moves towards the solution using A* search. (5 marks)

Solution 1.2:

a) Heuristic Function (5 marks)

  • Manhattan distance: Sum of horizontal and vertical distances each tile is from its goal position
  • h(n) = Σ |x₁ - x₂| + |y₁ - y₂| for each tile
  • Admissible because it never overestimates the actual cost

b) Manhattan Distance Calculation (5 marks)

Tile 2: |0-0| + |1-1| = 0
Tile 8: |0-1| + |1-0| = 2
Tile 3: |0-2| + |0-2| = 0
Tile 1: |1-0| + |0-0| = 2
Tile 6: |1-1| + |2-1| = 1
Tile 4: |1-2| + |1-2| = 0
Tile 7: |2-0| + |2-0| = 0
Tile 5: |2-2| + |2-2| = 0

Total Manhattan distance = 5

c) First Two A* Moves (5 marks)

  • Move 1: Slide 6 up
    • f(n) = g(n) + h(n) = 1 + 4 = 5
  • Move 2: Slide 8 down
    • f(n) = g(n) + h(n) = 2 + 3 = 5

2. Production System

Question 2.1 (15 Marks)

Design a production system for a weather advisory system with the following rules:

  • If temperature > 35°C and humidity > 70%, issue heat warning
  • If wind speed > 50 km/h and precipitation > 10mm, issue storm warning
  • If visibility < 100m and temperature < 0°C, issue winter storm warning

a) Write the production rules in formal IF-THEN format. (5 marks)

b) Create a conflict resolution strategy. (5 marks)

c) Show how the system would handle multiple simultaneous conditions. (5 marks)

Solution 2.1:

a) Production Rules (5 marks)

Rule 1:
IF (temperature > 35) AND (humidity > 70)
THEN warning = "Heat Warning"
      priority = HIGH
      action = "Issue public advisory"

Rule 2:
IF (wind_speed > 50) AND (precipitation > 10)
THEN warning = "Storm Warning"
      priority = CRITICAL
      action = "Issue emergency alert"

Rule 3:
IF (visibility < 100) AND (temperature < 0)
THEN warning = "Winter Storm Warning"
      priority = HIGH
      action = "Issue travel advisory"

b) Conflict Resolution (5 marks)

  • Priority Ordering:
    • CRITICAL: Immediate life threat
    • HIGH: Potential danger
    • MEDIUM: Caution needed
    • LOW: Advisory only
  • Specificity Principle: More specific conditions override general ones
  • Recency: Latest weather data takes precedence

c) Multiple Conditions Handling (5 marks)

Example Scenario:
Temperature = 36°C
Humidity = 75%
Wind_speed = 55 km/h
Precipitation = 12mm

System Response:
1. Rule 1 triggers (Heat Warning)
2. Rule 2 triggers (Storm Warning)
3. Conflict Resolution:
   - Storm Warning takes precedence (CRITICAL)
   - Heat Warning queued as secondary alert
4. Combined Advisory Issued

Question 2.2 (15 Marks)

Design a production system for a simple medical diagnosis system that can identify common cold, flu, and COVID-19 based on symptoms:

a) Create the knowledge base with at least 5 rules. (5 marks)

b) Draw the inference chain for a specific case. (5 marks)

c) Explain how certainty factors would be incorporated. (5 marks)

Solution 2.2:

a) Knowledge Base (5 marks)

Rule 1:
IF (fever < 38°C) AND (runny_nose = true) AND (sneezing = true)
THEN diagnosis = "Common Cold" (CF = 0.7)

Rule 2:
IF (fever >= 38°C) AND (body_aches = true) AND (fatigue = true)
THEN diagnosis = "Flu" (CF = 0.8)

Rule 3:
IF (fever >= 38°C) AND (loss_of_taste = true) AND (dry_cough = true)
THEN diagnosis = "COVID-19" (CF = 0.9)

Rule 4:
IF (contact_with_covid = true) AND (fever = true)
THEN diagnosis = "COVID-19" (CF = 0.6)

Rule 5:
IF (seasonal_allergies = true) AND (runny_nose = true)
THEN diagnosis = "Allergies" (CF = 0.5)

b) Inference Chain Example (5 marks)

Patient Symptoms:
- Fever 38.5°C
- Loss of taste
- Dry cough
- Fatigue

Inference Chain:
1. Rule 2 triggers (Flu possibility)
   CF = 0.8 * 0.7 = 0.56
2. Rule 3 triggers (COVID-19 possibility)
   CF = 0.9 * 0.9 = 0.81
3. System concludes COVID-19 due to higher CF

c) Certainty Factors Implementation (5 marks)

  • CF Calculation:
    • Each symptom has individual CF
    • Rule CF multiplied by symptom CFs
    • Combined CF for multiple rules
    • Threshold CF for diagnosis
  • Formula: CF(combined) = CF1 + CF2(1 - CF1) for similar conclusions
  • Minimum CF threshold = 0.5 for valid diagnosis

3. Problem Characteristics

Question 3.1 (15 Marks)

For the N-Queens problem with N=4:

a) Define all problem characteristics. (5 marks)

b) Write a state validity checking function. (5 marks)

c) Analyze the complexity of the search space. (5 marks)

Solution 3.1:

a) Problem Characteristics (5 marks)

  • Initial State: Empty 4x4 board
  • Goal State: 4 queens placed with no conflicts
  • Constraints:
    • No two queens on same row
    • No two queens on same column
    • No two queens on same diagonal
  • State Space: All possible queen arrangements
  • Operators: Place queen on empty square

b) State Validity Function (5 marks)

function isValid(board):
    n = 4
    for i in range(n):
        for j in range(i+1, n):
            # Check row and column
            if board[i][0] == board[j][0] or 
               board[i][1] == board[j][1]:
                return false
            
            # Check diagonals
            if abs(board[i][0] - board[j][0]) ==
               abs(board[i][1] - board[j][1]):
                return false
    return true

c) Complexity Analysis (5 marks)

  • Total possible arrangements: 16C4 = 1820
  • Valid solutions: 2 (considering rotations and reflections)
  • Time complexity: O(n²) for validity checking
  • Space complexity: O(n) for board representation

Question 3.2 (15 Marks)

For the Traveling Salesman Problem with 4 cities:

a) Define the problem characteristics and constraints. (5 marks)

b) Create a heuristic function for the problem. (5 marks)

c) Analyze why this is an NP-hard problem. (5 marks)

Solution 3.2:

a) Problem Characteristics (5 marks)

  • State Space: All possible city permutations
  • Initial State: Starting city
  • Goal State: Complete tour returning to start
  • Constraints:
    • Visit each city exactly once
    • Return to starting city
    • Minimize total distance

b) Heuristic Function (5 marks)

  • Minimum Spanning Tree (MST) Heuristic:
    • h(n) = cost of MST through unvisited cities + minimum edge to nearest unvisited city
    • Example calculation for 4 cities: 
              Cities: A, B, C, D
        Current Path: A → B
        h(n) = MST(C,D) + min(dist(B,C), dist(B,D))
  • Nearest Neighbor Heuristic:
    • h(n) = distance to closest unvisited city
    • Less accurate but faster to compute
  • Properties:
    • Admissible: Never overestimates actual cost
    • Consistent: Satisfies triangle inequality
    • Computationally efficient for small n

c) NP-Hard Analysis (5 marks)

  • Computational Complexity:
    • For n cities: (n-1)! possible tours
    • 4 cities = 6 possible tours (3!)
    • 5 cities = 24 possible tours (4!)
    • Growth is factorial: O(n!)
  • Why it's NP-Hard:
    • No known polynomial-time algorithm exists
    • All known exact solutions require exponential time
    • Can't be reduced to simpler problem
    • Verification is polynomial (P) but finding solution is NP
  • Example with 4 cities: 
            Cities: A, B, C, D
        Possible Tours:
        1. A → B → C → D → A
        2. A → B → D → C → A
        3. A → C → B → D → A
        4. A → C → D → B → A
        5. A → D → B → C → A
        6. A → D → C → B → A
        
        Total combinations = 6
        Checking each requires O(n) time
        Total complexity = O(n × n!)
  • Implications:
    • Must use approximation algorithms for large instances
    • Exact solutions only feasible for small n
    • Heuristic approaches necessary for practical applications

Important Note: For exam purposes, focus on:

  • Being able to calculate complexity for small instances
  • Understanding why the problem is NP-hard
  • Knowing how to apply heuristics
  • Explaining the exponential growth nature
