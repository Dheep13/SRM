Understanding O(b^m) Notation

Notation Components:

  • O (Big O): Represents the upper bound of growth rate
    • Describes worst-case scenario
    • Used to express maximum time/space needed
  • b (Branching Factor):
    • Number of successors (children) for each node
    • Number of possible moves/actions at each state
  • m (Maximum Depth):
    • Length of the longest path in the search tree
    • Maximum number of moves from start to goal

Example: 8-Puzzle

  • b (Branching Factor):
    • Each state has 2-4 possible moves (up, down, left, right)
    • Average b ≈ 3
  • m (Maximum Depth):
    • Worst case path length to solution
    • For 8-puzzle, m = 31 moves maximum

Complexity Calculation

For O(b^m):

  • Level 1: b nodes
  • Level 2: b² nodes
  • Level 3: b³ nodes
  • ...
  • Level m: b^m nodes

Example Values:

If b = 3 and m = 4:
Level 1: 3¹ = 3 nodes
Level 2: 3² = 9 nodes
Level 3: 3³ = 27 nodes
Level 4: 3⁴ = 81 nodes
Total nodes = 120 nodes

Real-World Examples

1. Chess Game:

  • b ≈ 35 (average legal moves)
  • m ≈ 100 (typical game length)
  • Complexity: O(35¹⁰⁰)

2. Rubik's Cube:

  • b = 18 (possible moves)
  • m = 20 (God's number)
  • Complexity: O(18²⁰)

3. Route Finding:

  • b = number of connected roads
  • m = maximum route length
  • Complexity varies with map size

Important Points for Exams:

  • Can you identify b and m for a given problem?
  • Can you calculate total nodes at each level?
  • Can you compare different algorithms' complexities?
  • Common complexities in search algorithms:
    • BFS: O(b^d) where d is solution depth
    • DFS: O(b^m) where m is maximum depth
    • A*: O(b^d) but usually much better in practice

Practice Questions:

  1. For a puzzle with 4 possible moves at each state and maximum solution depth of 10, what is the worst-case complexity?
  2. If each node in a search tree has 3 children and the solution is at depth 5, how many nodes might need to be examined in the worst case?
  3. Compare the complexities of searching a tree with b=2, m=10 versus b=3, m=5.

Answers:

  1. O(4¹⁰) ≈ 1,048,576 nodes
  2. O(3⁵) = 243 nodes
  3. 2¹⁰ = 1,024 vs 3⁵ = 243 (second is smaller)
